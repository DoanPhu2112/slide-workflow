Flow tổng quát

4 bước phải đi đúng thứ tự

Nếu đi ngược thứ tự, bạn sẽ phải quay qua lại giữa dashboard free domain và Cloudflare mà không rõ chỗ nào đang là nguồn cấu hình thật.

01

Tạo account

Create account trong DigitalPlat, rồi mới liên kết GitHub nếu cần.

Output: có dashboard quản lý domain
02

Register domain

Nhập subdomain, chọn đuôi miền free, kiểm tra availability và lấy slot.

Output: domain đã được cấp
03

Chuyển DNS

Add domain vào Cloudflare, copy nameserver và cập nhật ngược lại.

Output: Cloudflare quản lý DNS
04

Trỏ website

Thêm A record về server/hosting, bật SSL mode đúng và chờ propagate.

Output: web mở được từ internet
Chuẩn bị

Google account, Cloudflare account và 1 public IPv4 hoặc hosting free để test.

Điểm dễ nhầm

Nameserver phải đổi xong trước khi record trong Cloudflare có ý nghĩa.

Bước 1

Tạo account DigitalPlat đúng cách

Bài gốc lưu ý không nên bấm login bằng Google hay GitHub ngay ở bước đầu. Hãy tạo account thường, sau đó mới liên kết tài khoản trong dashboard.

  1. Truy cập https://dash.domain.digitalplat.org/auth/login.
  2. Chọn Create an account.
  3. Hoàn tất account thường để vào dashboard.
  4. Nếu muốn, liên kết thêm GitHub sau khi đã đăng nhập.
Lý do: tránh rối flow xác thực ngay từ đầu và giữ quyền chủ động ở dashboard domain.
điều kiện đầu vào

Chỉ cần 1 Google account

Phần quản lý DNS chuyên nghiệp sẽ được đẩy sang Cloudflare ở bước tiếp theo, nên DigitalPlat chỉ là điểm cấp phát domain free ban đầu.

Bước 2

Chọn subdomain và lấy free slot

Đây là bước xác định domain cuối cùng bạn sẽ mang qua Cloudflare. Ví dụ trong bài là devopsvietnamlab.dpdns.org.

  1. Nhập subdomain mong muốn, ví dụ devopsvietnamlab.
  2. Chọn một đuôi miền free, ví dụ dpdns.org.
  3. Bấm Check availability.
  4. Nếu còn trống, chọn Use Free Slot rồi Register.
Lưu ý: phần nameserver có thể để xử lý ngay sau đó khi bạn đã có thông tin từ Cloudflare.
domain example
subdomain: devopsvietnamlab
suffix:    dpdns.org
result:    devopsvietnamlab.dpdns.org

Done khi nào?

  • Domain đã xuất hiện trong dashboard.
  • Slot type đang là free slot.
  • Bạn có đúng full domain để đem qua bước Cloudflare.

Bước 3

Chuyển DNS sang Cloudflare

Từ bước này trở đi, Cloudflare mới là nơi bạn quản lý A, CNAME, TXT và SSL.

  1. Đăng nhập https://dash.cloudflare.com/.
  2. Vào Domains → Add domain → Connect a domain.
  3. Nhập full domain vừa đăng ký ở DigitalPlat.
  4. Chọn plan Free.
  5. Vào DNS → Settings để lấy 2 nameserver.
  6. Quay lại DigitalPlat và cập nhật 2 nameserver đó cho domain.
handoff rule
DigitalPlat
  cấp domain free

Cloudflare
  cấp nameserver
  quản lý DNS records
  quản lý SSL/TLS mode
checkpoint

2 nameserver đã được cập nhật ngược lại

Nếu bạn chưa làm xong bước này, mọi record bạn thêm trong Cloudflare vẫn chưa phải nguồn DNS có hiệu lực cho domain đó.

Bước 4

Trỏ website về server và cấu hình SSL/TLS

Sau khi nameserver đã chuyển xong, bạn mới thêm record để đẩy web lên internet thật.

  1. Vào Cloudflare → DNS → Records.
  2. Thêm bản ghi A với Name = @.
  3. Trỏ về your_server_IP_address.
  4. Bật proxy nếu muốn đi qua edge của Cloudflare.
  5. Nếu site dùng HTTPS, đặt SSL/TLS mode là Full hoặc Full (strict).
minimum record
Type:         A
Name:         @
IPv4 address: your_server_IP_address
Proxy status: Enable
Cảnh báo: SSL/TLS mode sai có thể gây redirect loop hoặc lỗi SSL dù DNS đã đúng.

Checklist cuối

Kiểm tra nhanh trước khi kết luận là xong

DigitalPlat

  • Domain đã register thành công.
  • 2 nameserver đã đổi sang Cloudflare.
  • Không tiếp tục sửa record tại đây.

Cloudflare

  • Domain đã được add vào dashboard.
  • A record trỏ đúng đến IP đích.
  • SSL/TLS mode phù hợp với server.

Lỗi thường gặp

  • Subdomain đã bị dùng trước đó.
  • Quên đổi nameserver.
  • Trỏ sai IP hoặc sai SSL mode.
